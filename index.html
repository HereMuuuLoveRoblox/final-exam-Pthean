⚠️ ข้อสังเกต / ปัญหาที่พบ 7 รายการ
🔴 HIGH  —  ไม่มี Authentication จริง
API ทุกตัวใช้แค่ user_id ใน Body โดยไม่มี JWT Token ผู้ใช้สามารถส่ง user_id ของคนอื่นเพื่อดูข้อมูลได้ ควรเพิ่ม JWT Bearer Token ใน Authorization Header
🔴 HIGH  —  Backend 2 ชุดซ้อนกัน — checkout ไม่ตรงกัน
API endpoint/orders/checkout.php รับแค่ {"user_id"} (ใหม่)
ebook-store/api/orders/checkout.php รับ {"user_id","total_price","cart_items[]"} (เก่า)
Frontend ส่งแค่ user_id → ต้องใช้ไฟล์ใหม่เท่านั้น
🟡 MEDIUM  —  ebook-store/api/ ขาดไฟล์หลายตัว
ไฟล์ต่อไปนี้มีแค่ใน API endpoint/ แต่ยังไม่ได้ deploy ไป ebook-store/api/:
cart/add.php · cart/list.php · cart/remove.php · orders/get_order_details.php · orders/get_pending_orders.php · payments/submit_payment.php · ebooks/my_ebooks.php
🟡 MEDIUM  —  Trailing Slash ใน OrdersScreen URL
const API_ORDERS_URL = ".../get_orders.php/";
trailing slash อาจทำให้ server คืน 404 ขึ้นอยู่กับ Apache/Nginx config
🔵 LOW  —  Logout API ไม่ถูกเรียก
Frontend logout ผ่าน AsyncStorage.removeItem("user") อย่างเดียว ไม่ได้เรียก /auth/logout.php → PHP session ฝั่ง server ยังค้างอยู่
🔵 LOW  —  clearCart() N+1 Problem
CartScreen.js ล้างตะกร้าโดยวน axios.post(remove) ทีละ item แทนที่จะใช้ API DELETE FROM cart WHERE user_id = ? ทีเดียว
🔵 LOW  —  ไม่มี Admin REST API
Admin functions (confirm_payment, manage_ebooks, manage_users) ทำผ่าน PHP Web Pages โดยตรง ไม่สามารถเรียกจาก Mobile App ได้ หากต้องการ Admin App ในอนาคตต้องสร้าง Admin REST API แยก
